# LearningGitHub 项目简介（产品视角）

> 文档版本：v0.1（MVP 冻结版，后续如有较大改动请通过新版本维护）

## 一句话定位

LearningGitHub 是一个「帮你看懂任意 GitHub 项目」的中文学习助手。用户只要粘贴 GitHub 仓库链接，就能自动得到用自己的母语写成的、手把手的项目学习教程。

## 目标用户

- 想用开源项目、但读不懂英文 README 和文档的中文用户
- 会基本命令行 / Git 操作，但对复杂项目结构感到困难的初学者
- 想快速理解一个项目做什么、如何跑起来的产品/运营/学生

## 用户痛点

- GitHub 页面以英文为主，看不懂说明和 Issue
- 项目目录复杂，不知道「第一步应该看哪个文件、做什么操作」
- 官方文档往往面向有经验的开发者，不适合完全新手
- 想「跟着做」完成一个功能，却缺少从 0 到 1 的详细教程

## 产品核心价值

- 将任意 GitHub 仓库转化为「中文 + 分步讲解」的教程
- 告诉用户：这个项目是做什么的、适合谁用、需要哪些前置知识
- 给出「从克隆仓库到运行成功」的完整步骤和常见问题排查
- 支持用户用自然语言提问「这个文件/函数是干什么的」并获得解释

## MVP 核心功能

> 以「最低可用产品」为目标，优先实现最能产生学习价值的部分。

- **仓库解析**
  - 用户输入 GitHub 仓库 URL
  - 系统抓取 README、主要代码文件和目录结构，生成项目概览

- **项目中文概览**
  - 自动生成简洁的中文介绍：项目用途、主要功能、技术栈
  - 用非专业用户能理解的语言解释「这个项目能帮你做什么」

- **手把手运行教程**
  - 从「环境准备」到「启动项目」给出详细步骤
  - 每一步都包含：要执行的命令、预期结果、遇到错误怎么办

- **文件与模块讲解（基础版）**
  - 为核心目录和关键文件生成简要说明：这个目录/文件负责什么
  - 帮助用户建立项目整体结构的心智模型

- **项目拆分与学习路径（基础版）**
  - 将项目按主题或功能拆成若干学习模块（如「环境准备」「核心功能」「扩展功能」）
  - 为每个模块生成简要说明与推荐学习顺序，支持在界面中展示模块列表与完成状态

- **中文为主的多语言支持**
  - 默认输出为简体中文
  - 后续可以配置输出为英文或其他语言（非 MVP 强制）

## 用户使用流程（MVP）

1. 打开网页或应用，粘贴一个 GitHub 仓库 URL  
2. 选择教程语言（默认中文），点击「开始学习」  
3. 系统解析仓库，生成项目概览、项目结构/文件说明、运行步骤和学习模块列表  
4. 用户按步骤操作，遇到问题时可以在页面里提问  
5. 按模块化学习路径完成「从 0 到项目跑起来」的整个学习过程

## 不在 MVP 范围内的功能（未来可选）

- 游戏化元素（成就、勋章、学习时长统计等）
- 社区讨论区和学习笔记分享
- 本地部署大型语言模型、离线使用等高级能力

这些功能可以根据真实用户反馈和项目进展，分阶段逐步加入，而不阻碍当前 MVP 版本的快速落地与验证。
