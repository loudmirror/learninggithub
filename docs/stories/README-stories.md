## 📝 Story 模板说明

每个 Story 包含以下标准部分：

1. **元信息**: Story ID, Epic, 迭代, 状态, 优先级, 预估时间
2. **User Story**: As a... I want... So that...
3. **背景与上下文**: 项目上下文、技术上下文、迭代目标
4. **验收标准**: 功能性需求 + 质量需求
5. **技术实现任务**: 详细的任务清单（可执行）
6. **技术说明**: 代码示例、配置、命令
7. **风险与依赖**: 风险评估、依赖关系
8. **Definition of Done**: 完成定义检查清单
9. **Dev Agent Record**: 开发记录区域
10. **相关文档**: 链接到相关资源
