# LearningGitHub UX 核心流程设计（MVP）

> 文档版本：v0.1（MVP 冻结版，后续修改请在新版本中进行）

## 1. 核心用户流程

### 流程 A：第一次使用，学习一个新仓库（主流程）

1. 用户打开首页。
2. 看到「粘贴 GitHub 仓库 URL」的主输入框与简短说明。
3. 粘贴 URL，选择语言（默认中文），点击「开始学习」按钮。
4. 系统校验 URL（格式/是否 github.com 域名）。
5. 校验通过后进入「解析中」状态，展示进度提示。
6. 解析完成后跳转到「教程页面」，展示：
   - 项目概览（项目做什么）。
   - 项目结构与文件讲解（核心目录/关键文件负责什么）。
   - 运行前提条件（环境要求，如语言版本、工具依赖）。
   - 模块化学习路径视图（模块列表及推荐顺序）。
   - 当前模块的分步运行教程。
7. 用户按步骤执行命令，对照预期结果完成项目运行。
8. 遇到问题时，在教程页面的问答区域输入问题获取解答。

### 流程 B：回到之前学过的仓库

1. 用户再次进入首页。
2. 首页展示「最近学习的项目」列表（名称 + 语言 + 星数）。
3. 用户点击其中一个项目卡片。
4. 直接进入对应仓库的教程页面（缓存存在则复用结果）。

### 流程 C：教程页面内问答

1. 用户在教程中遇到不理解的步骤或概念。
2. 滚动到问答区域，在输入框中用自然语言提问（中文为主）。
3. 系统显示「正在思考/生成中」状态。
4. 回答生成后展示在对话历史中，用户可继续追问。

### 流程 D：异常/错误流程

1. URL 非 GitHub 或格式错误 → 首页输入框下方即时红字提示，禁止提交。
2. 仓库不存在 / 私有 / API 出错 → 在「解析中」状态后显示错误信息和重试/返回首页按钮。
3. 仓库过大或类型不支持 → 提示「项目过大/暂不支持」，建议用户换仓库。

## 2. 关键页面/视图

### 2.1 首页（Home）

- 目标：让用户快速理解产品，并开始输入第一个 GitHub URL。
- 内容结构：
  - 顶部：一句话说明（例如「粘贴 GitHub 链接，生成中文手把手教程」）。
  - 中部：
    - GitHub URL 输入框。
    - 语言选择器（默认中文）。
    - 「开始学习」主按钮。
  - 下部：
    - 最近学习项目列表（如有）。
    - 简短「如何使用」三步说明。

### 2.2 解析中（Loading 状态）

- 可作为独立页面或教程页面的覆盖层。
- 展示：
  - 当前阶段（例如「获取仓库信息」「分析代码结构」「生成教程」）。
  - 提示解析通常需要 30–90 秒。
  - 若失败，切换到错误提示模块。

### 2.3 教程页面（Tutorial View）

- 顶部信息条：
  - 项目名称、star 数、主要语言、GitHub 链接。
  - 返回首页或切换项目的入口。
- 左侧/上方：学习内容区域
  - 模块 1：项目概览（用途、功能、适用人群、技术栈）。
  - 模块 2：项目结构与文件讲解（核心目录/关键文件的简要说明）。
  - 模块 3：运行前准备（环境要求与必要工具）。
  - 模块 4：模块化学习路径视图（模块列表、顺序及完成状态指示）。
  - 模块 5：当前模块的分步运行教程：
    - 每一步是可展开/折叠的卡片。
    - 含命令代码块、说明、预期结果和常见错误排查。
- 右侧/下方：问答区域
  - 对话历史列表。
  - 问题输入框 + 提交按钮。
  - 顶部标签显示当前仓库信息。

### 2.4 错误/异常视图

- 统一模板：
  - 错误标题（如「无法解析这个仓库」）。
  - 错误原因说明。
  - 建议的下一步操作（重试、换仓库、稍后再试）。
  - 返回首页按钮。

## 3. 关键交互要素

### 3.1 URL 输入与校验

- 即时校验 URL：
  - 非 GitHub 格式时在输入框下方显示错误提示。
  - 无效时禁用提交按钮。
- 自动清理：
  - 去除首尾空格和多余斜杠，减少用户操作负担。

### 3.2 步骤式教程

- 每个步骤：
  - 支持展开/折叠。
  - 有「标记为完成」操作，改变步骤视觉状态。
  - 命令代码块带「一键复制」按钮并显示「已复制」反馈。

### 3.3 问答区域

- 输入：
  - 支持回车发送及显式「提问」按钮。
  - 提交后显示加载状态，避免重复提交。
- 输出：
  - 回答自动追加到对话历史末尾。
  - 界面自动滚动显示最新回答。

### 3.4 状态与反馈

- 加载状态：解析与生成时显示进度提示与简短说明。
- 成功状态：在教程完成关键步骤后给予简短鼓励文案。
- 错误状态：使用友好文案解释原因并给出下一步建议。
