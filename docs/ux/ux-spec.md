# LearningGitHub UX 规格说明（MVP）

> 文档版本：v0.1（对应 PRD v0.1，架构 v0.1）  
> 关联文档：`docs/prd.md`, `docs/architecture.md`, `docs/ux/ux-core-flow.md`, `docs/ux/ux-pages-and-layout.md`, `docs/ux/front-end-spec.md`

---

## 1. UX 目标与原则

- 降低普通中文用户理解 GitHub 项目的门槛，让「看不懂英文 README」「不知从哪开始」的人可以按教程完成一个项目的学习与跑通。
- 核心体验目标：
  - 用户在 1 分钟内看懂项目是做什么的、适合谁。
  - 用户可以按步骤在本地跑起来项目，并知道自己当前在学习路径中的位置。
  - 用户遇到疑问时，能自然地在页面内提问并获得上下文相关回答。
- 设计原则：
  - 渐进披露：先给高层概要，再逐步展开结构、模块和具体步骤。
  - 保持轻量：页面信息密度控制在「大卡片 + 明显留白」水平，避免拥挤。
  - 一致性：桌面与移动端在信息顺序、组件风格上保持一致，仅布局适配。

---

## 2. 支持平台与断点

- 桌面端：视口宽度 ≥ 1024px，主设计参考宽度 ~1440px。
- 移动端：视口宽度 < 1024px，采用单列布局，教程内容优先、问答通过 Tab 或切换显示。
- 响应式规则见 `docs/ux/front-end-spec.md` 第 8 节。

## 3. 核心用户流程（摘要）

> 完整流程细节见 `docs/ux/ux-core-flow.md`。

1. 首次使用（主流程）  
   - 进入首页 → 粘贴 GitHub URL → 选择语言 → 点击「开始学习」。  
   - 系统解析仓库并生成教程 → 跳转到教程页面。  
   - 用户按模块化学习路径完成各模块的关键步骤 → 跑通项目 → 可继续探索或结束。

2. 回到已学项目  
   - 进入首页 → 在「最近学习的项目」中选择 → 直接打开对应教程页面。

3. 教程内问答  
   - 在教程页面遇到不理解的步骤/结构 → 在右侧/底部问答区域以自然语言提问 → 获得回答 → 可继续追问。

4. 异常流程  
   - URL 不合法/仓库不存在/仓库超限/GitHub 限流时提供清晰错误提示和下一步建议（返回首页/重试/更换仓库）。

## 4. 页面规格

### 4.1 首页（Home）

- 路由：`/`
- 目标：让用户立即理解产品用途，并开始输入第一个 GitHub URL。
- 主要区域：
  - Header：产品名「LearningGitHub」+ 简短副标题（例如「看懂每一个 GitHub 项目」）。
  - 主表单区：
    - GitHub URL 输入框（含示例 placeholder 和基础校验）。
    - 语言选择器（默认简体中文）。
    - 主按钮「开始学习」。
    - 辅助说明：支持公开仓库，解析时间约 30–90 秒等。
  - 最近学习项目区：
    - 列表/卡片形式展示最近几个仓库（名称、owner、language、stars）。
    - 每个卡片有「继续学习」入口。
  - 使用说明区（简短三步）：
    - 粘贴链接 → 等待解析 → 跟着教程跑起来。
- 状态：
  - Loading：提交 URL 后，按钮进入 loading 状态，避免重复提交。
  - 错误：URL 格式错误或非 GitHub 域名时在输入框下显示错误文案，按钮禁用。

### 4.2 教程页面（Tutorial）

- 路由：`/tutorial/:owner/:repo`（可含 query repoUrl）。
- 页面布局（桌面端）：
  - 顶部：
    - 仓库名称 `owner/repo`，带 GitHub 链接按钮。
    - 主要语言、stars 等 meta 信息。
    - 返回首页入口。
  - 主内容区：左右两列布局
    - 左列（学习内容）：
      1. **项目概览模块**：  
         - 简短描述项目用途、适合人群、主要技术栈。  
      2. **项目结构与重要文件模块**：  
         - 列表/紧凑手风琴显示核心目录和关键文件，一行说明其职责。  
      3. **运行前准备模块**：  
         - 列表形式说明运行项目前需要安装/准备的环境（语言版本、依赖包管理器、必备工具等）。  
         - 作为从「了解项目」到「实际动手」之间的过渡层。  
      4. **学习路径模块**：  
         - 模块列表（环境准备、跑通项目、理解核心功能等）。  
         - 显示当前模块、已完成模块和整体进度。  
      5. **当前模块的运行步骤模块**：  
         - 仅展示当前模块的关键步骤（标题 + 简短说明 + 命令/链接 + 预期结果）。  
    - 右列（问答区域）：
      - 标题「AI 学习助手」。  
      - 聊天历史列表（最近若干条）。  
      - 问题输入框 + 发送按钮，含 placeholder 文案（如「有问题就问我吧…」）。
- 页面布局（移动端）：
  - 顶部结构同桌面端。
  - 主内容区改为单列纵向，模块顺序与桌面一致：
    - 项目概览 → 项目结构与文件讲解 → 学习路径 → 当前模块步骤。  
  - 问答区域通过 Tab 或底部入口显示：  
    - Tab1：「教程」  
    - Tab2：「问答」

## 5. 交互与状态规格

### 5.1 学习路径与步骤

- 学习路径模块列表：
  - 点击模块项 → 将该模块设为当前模块：  
    - 模块项高亮；  
    - 当前模块步骤模块内容更新；  
    - 页面滚动到当前模块步骤区域。
  - 模块项显示状态：
    - 未开始：默认样式。  
    - 当前：高亮边框或背景。  
    - 已完成：勾选图标或颜色变化。
- 当前模块步骤：
  - 每个步骤可标记为完成（复选框或勾选按钮）。  
  - 完成后步骤可淡化或标记为「已完成」。  
  - 命令部分提供「复制」按钮和复制成功短提示。  

### 5.2 问答区域

- 输入：
  - 支持回车发送和点击按钮发送。  
  - 空内容禁止发送，并给出轻微提示（例如输入框边框闪动）。  
- 输出：
  - 提交后立即显示「思考中…」或类似 loading 消息。  
  - 回答气泡插入到对话历史末尾，自动滚动显示。  
  - 支持多轮对话，保持与当前仓库上下文绑定。

### 5.3 Loading 与错误状态

- Loading：
  - 解析教程数据：展示页面骨架或通用加载提示（如「正在解析仓库并生成教程…」）。  
  - 问答：仅在问答区域显示局部加载，不影响其他操作。
- 错误：
  - 教程加载失败：使用统一错误组件显示错误原因（仓库不存在、超出限制、GitHub 限流等）+「返回首页」/「重试」按钮。  
  - 问答失败：在对话历史中追加一条错误系统消息（如「当前无法获取回答，请稍后再试」）。

## 6. 文案风格与本地化

- 默认语言：简体中文。  
- 文案风格：
  - 使用「你」而不是「您」，语气友好、鼓励式。  
  - 避免堆砌专业术语，对必要术语配一句短解释。  
  - 在提示错误或风险时，给出明确的下一步建议，而不是仅陈述问题。  
- 国际化：
  - 所有用户可见文案应集中在可配置文本中，避免硬编码在逻辑层。  
  - 后续扩展多语言时，优先基于文案 key 和翻译表实现。

## 7. 可用性与后续优化方向

MVP 可用性关注点：

- 用户是否能在 1–2 分钟内理解项目用途与学习路径。  
- 用户是否能顺利跟随教程跑通一个中小型项目。  
- 用户在学习过程中是否会迷失当前模块或进度。

后续优化（非本版范围）：

- 更精细的模块拆分与个性化推荐学习顺序。  
- 游戏化元素（进度条、成就、勋章）。  
- 更丰富的项目结构可视化（树状图/关系图）。
