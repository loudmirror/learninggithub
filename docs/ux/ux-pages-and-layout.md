# LearningGitHub 关键页面与布局说明（MVP）

> 文档版本：v0.1（MVP 冻结版，后续布局变更请通过新版本维护）

## 1. 首页（Home）

### 1.1 布局结构

- 顶部区域：
  - 产品标题（例：「LearningGitHub」）。
  - 一句话说明（例：「粘贴 GitHub 链接，生成中文手把手教程」）。
- 中央主区域：
  - GitHub URL 输入框（支持粘贴和输入）。
  - 语言选择器（下拉，默认简体中文）。
  - 「开始学习」主按钮（显眼主色）。
- 下方辅助区域：
  - 最近学习项目列表（卡片形式，显示仓库名称、语言、star 数）。
  - 简短「如何使用」三步说明。

### 1.2 关键交互

- URL 输入框：
  - 失焦或输入变化时进行基础校验。
  - 错误时显示红色提示文案并禁用提交。
- 「开始学习」按钮：
  - 正常状态可点击，loading 时显示转圈并禁用。
- 最近项目卡片：
  - 点击跳转到对应教程页面。

---

## 2. 教程页面（Tutorial View）

### 2.1 布局结构（桌面）

- 顶部栏：
  - 仓库名称 + GitHub 图标链接。
  - star 数、主要语言等基本信息。
  - 返回首页按钮。
- 主内容区：左右布局
  - 左侧（或上方）：学习内容
    - 模块一：项目概览。
    - 模块二：项目结构与文件讲解（核心目录/关键文件列表及说明）。
    - 模块三：运行前准备（环境要求与必备工具列表）。
    - 模块四：模块化学习路径视图（模块列表、顺序与完成进度指示，可点击跳转到对应内容区域）。
    - 模块五：当前模块的分步运行教程（步骤列表）。
  - 右侧（或下方）：问答区域
    - 对话历史。
    - 问题输入框 + 提交按钮。

### 2.2 布局结构（移动）

- 顶部栏同桌面版。
- 主内容区：
  - 采用「选项卡」或切换按钮在「教程」和「问答」间切换。
  - 优先展示教程内容，将问答放在第二标签页。

### 2.3 关键交互

- 步骤列表：
  - 每个步骤可折叠/展开。
  - 每个步骤有小复选框/按钮标记完成，完成后淡化或打勾。
- 学习路径模块列表：
  - 模块项可点击，高亮当前模块并滚动/跳转到对应教程内容。
  - 模块项支持「已完成」标记，整体学习进度通过已完成模块数体现。
- 命令块：
  - 支持一键复制。
  - 显示短暂提示「已复制」。
- 问答区：
  - 输入后按回车或点击按钮发送。
  - 显示模型推理中的状态（例如「正在生成回答…」）。

## 3. 解析中与错误状态

### 3.1 解析中

- 显示内容：
  - 简短说明（如「正在解析仓库并生成教程，这通常需要 30–90 秒」）。
  - 可选进度阶段提示（如「获取仓库信息」「分析代码结构」「生成教程内容」）。
- 行为：
  - 禁用主要交互按钮，避免中途重复发起请求。

### 3.2 错误状态

- 统一错误组件：
  - 错误图标 + 标题（如「无法解析该仓库」）。
  - 具体原因简述。
  - 「返回首页」按钮。
  - （可选）「重试」按钮。

---

## 4. 文案与语气（MVP）

- 语气友好、鼓励式，避免过多技术术语。
- 对于重要概念，尽量用一句话解释用途。
- 尤其在错误提示和教程步骤中，强调「下一步可以怎么做」。
